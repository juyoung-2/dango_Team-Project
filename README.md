# DanGo

일본어 단어 학습과 커뮤니티 기능을 결합한 웹 서비스입니다.  
사용자는 새로운 단어를 학습하고, 오답노트와 학습 기록을 통해 반복 복습할 수 있으며, 커뮤니티를 통해 학습 정보와 경험을 공유할 수 있습니다.

---

## 프로젝트 소개

**DanGo**는 일본어 학습을 보다 쉽고 꾸준하게 이어갈 수 있도록 기획한 웹 프로젝트입니다.  
단어 학습, 오답노트, 단어장, 사전, 커뮤니티 기능을 하나의 서비스 안에 구성하여 학습과 소통이 함께 이루어지도록 설계했습니다.

메인 화면에서는 새 단어 목록과 커뮤니티 최신 글을 한눈에 확인할 수 있도록 구성했으며,  
학습 기능과 게시판 기능을 자연스럽게 연결한 점이 특징입니다.

## 주요 기능

- **새 단어 학습**  
  일본어 단어, 후리가나, 뜻을 함께 제공하여 카드 형태로 학습할 수 있습니다.

- **오답노트**  
  틀린 단어를 별도로 관리하여 반복 복습이 가능하도록 구성했습니다.

- **단어장 / 사전**  
  학습 중 필요한 단어를 모아 보거나 검색하며 확인할 수 있습니다.

- **학습 기록 관리**  
  체크한 단어, 학습 완료 단어, 최근 학습 단어, 오답 단어를 기록하여 학습 흐름을 관리할 수 있습니다.

- **커뮤니티**  
  게시글, 댓글, 대댓글, 추천, 스크랩 기능을 통해 학습 관련 정보를 공유하고 소통할 수 있습니다.

- **회원 기능**  
  로그인 / 회원가입을 통해 사용자별 학습 기록과 커뮤니티 활동을 관리할 수 있습니다.

## 프로젝트 구조

프로젝트는 Controller - Service - DAO - Mapper 구조를 기반으로 구성했으며,  
단어 학습과 커뮤니티 기능을 패키지 단위로 분리해 관리했습니다.

```text
dango
├─ src/main/java
│  └─ com.threeteam.dango
│     ├─ controller
│     ├─ dao
│     ├─ domain
│     ├─ mapper
│     ├─ service
│     └─ vo
├─ src/main/resources
├─ src/main/webapp
├─ pom.xml
└─ README.md
```

## 트러블슈팅

### GitHub 업로드 시 상위 폴더 전체가 함께 업로드되는 문제

프로젝트를 GitHub에 업로드하는 과정에서 `dango` 프로젝트만 올린다고 생각했지만, 상위 작업 폴더 전체가 함께 업로드되는 문제가 있었습니다.  
원인은 `dango` 폴더가 독립 Git 저장소가 아니라 상위 폴더의 Git 저장소 하위 경로로 포함되어 있었기 때문이었습니다.  
프로젝트 폴더를 별도 경로로 분리한 뒤 `dango` 폴더에서 새로 Git 저장소를 초기화하여 독립적으로 관리하도록 수정했고, 이후 프로젝트만 정상적으로 업로드할 수 있었습니다.

### 기능 테스트를 위한 더미 데이터 구성 과정에서의 정합성 문제

게시판과 단어 학습 기능을 테스트하기 위해 더미 데이터를 구성하는 과정에서, 단순히 데이터를 입력하는 것만으로는 화면과 기능이 자연스럽게 동작하지 않는 문제가 있었습니다.  
특히 게시글, 댓글, 추천과 같은 커뮤니티 데이터와 단어, 예문, 학습 기록 데이터는 각각의 기능 흐름에 맞게 연결되어야 테스트가 가능했습니다.  
이를 통해 더미 데이터 역시 단순한 샘플이 아니라, 실제 서비스 흐름을 검증할 수 있도록 구조적으로 구성되어야 한다는 점을 확인할 수 있었습니다.

## 회고

이번 프로젝트를 통해 단순히 화면을 구성하는 것에 그치지 않고, 데이터 구조와 화면 출력 흐름이 함께 맞물려야 기능이 완성된다는 점을 배울 수 있었습니다.  
특히 메인 화면에서 커뮤니티 최신글과 단어 학습 데이터를 연결하는 과정에서, 화면 구성과 데이터 조회가 하나의 흐름으로 이어져야 한다는 점을 체감했습니다.

또한 게시판과 단어 학습 기능 테스트를 위한 더미 데이터를 직접 구성하면서, 기능 구현만큼이나 테스트를 위한 데이터 설계와 초기 환경 정리의 중요성도 확인할 수 있었습니다.  
Spring MVC, MyBatis, Oracle 기반의 계층 구조를 직접 접하며 Controller, Service, DAO, Mapper가 어떤 역할로 나뉘고 연결되는지 구체적으로 이해할 수 있었고, GitHub 업로드와 README 작성까지 진행하면서 결과물을 문서화하고 정리하는 과정 역시 프로젝트의 중요한 일부라는 점을 배울 수 있었습니다.
